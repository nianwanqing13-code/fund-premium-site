## 一、这个项目是干什么的（一句话）

一个**网站**，实时计算并展示「场内基金（ETF / LOF）的溢价率」，并带几个实用功能：
1. **基金能不能申购**（申购状态：开放 / 限大额 / 暂停）
2. **基金能申购多少**（可申购数额：日累计申购限额，如 `1000.00元`、`50.00万元`）
3. **基金什么时间能申购**（申购时间提示）
4. **基金申购到账时间**（ETF 实物申赎 / LOF T+2）
- 另外，本次已**修复 LOF 数据缺失**问题（之前 LOF 一直是 0 只，现在能正常抓取约 370+ 只）。

溢价率 =（当日市价 − 当日预估净值 IOPV）÷ 预估净值 × 100%
- 溢价率 > 0（标红）：市价比净值贵，可能适合「卖出/赎回套利」
- 溢价率 < 0（标绿）：市价比净值便宜，可能适合「买入套利」

数据来源：[AKShare](https://github.com/akfamily/akshare)（东方财富公开行情）＋ 天天基金档案页（申购状态/限额），**全部免费、无需账号密钥**。

## 二、本次交接重点：五个功能 / 改动

### 功能 1：基金申购状态（能不能申购）
- **位置**：网站表格里的「申购状态」列。
- **显示四种情况，用颜色区分**：
  - 🟢 `开放申购·可申购`（绿色）—— 正常可买，不限额度
  - 🟠 `限大额·可申购`（橙色）—— 可买但有金额上限（上限见「可申购数额」列）
  - 🔴 `暂停申购（不可申购）`（红色）—— 当前不可买
  - ⚪ `ETF·实物申赎`（灰色）—— ETF 是实物申赎（需一篮子股票），天天基金的现金申赎状态对它没意义，故单列说明
  - 显示 `-` 表示首屏后台还在抓取（等约 1 分钟会自动补齐）
- **怎么实现的（重要，跟旧文档不同）**：
  - ⚠️ **没有用 AKShare 取申购状态**。AKShare 的 `fund_open_fund_info_em()` 其实返回的是「单只基金历史净值」，根本没有申赎状态字段，旧代码一直拿不到数据（列里全是 `-`）。这是之前的一个坑，已废弃。
  - 现在后端 `app.py` 的 `get_subscribe_map()` 直接抓**天天基金的「基金费率」档案页** `https://fundf10.eastmoney.com/jjfl_<代码>.html`，用正则提取「交易状态」和「日累计申购限额」。
  - 用 `_normalize_subscribe()` 把原始中文整理成上面四种状态。
  - **只给 LOF 取真实状态**：LOF 是场内现金申购，状态/限额对套利直接有用；ETF 是实物申赎，单独标注。
  - 数据带 **1 小时缓存**（`_SUBSCRIBE_TTL=3600`），且首次用**后台线程** ` _build_subscribe_map()` 慢慢抓取（约数百只 LOF，受天天基金节流影响要几十秒），接口先秒回已有数据，前端靠自动刷新自然补齐。

### 功能 2：可申购数额（能申购多少，新列 + 可排序）
- **位置**：表格里「申购状态」右侧新增的「可申购数额」列（功能 1 的状态文字已不含数额，数额单独放这一列）。
- **显示**：日累计申购限额原文，如 `1000.00元`、`50.00万元`、`5000.00元`；暂停申购 / ETF / 暂无数据时显示 `-`。
- **可排序**：点表头「可申购数额」即可按金额从小到大 / 从大到小排。
  - 实现：后端 `_parse_limit_to_num()` 把限额文字解析成**以「元」为单位的数值**（自动统一单位：`50.00万元`→500000、`2.50亿元`→250000000），随接口返回 `amount_num` 字段；前端排序时按该数值排，没有限额的排到末尾。

### 功能 3：基金申购时间（什么时间能申购）
- **位置**：网站顶部的一个提示条，实时显示「当前可申购 / 暂不可申购」。
- **怎么实现的**：
  - 申购时间**没有现成接口可查**，是固定规则：交易日（周一至周五，非法定节假日）的 **9:30–11:30、13:00–15:00** 才能申购。
  - 后端 `app.py` 的 `is_trade_time()` 按当前时间判断，前端提示条显示结果。

### 功能 4：基金申购到账时间
- **位置**：表格里「可申购数额」右侧的「到账时间」列。
- **显示规则**（前端 `arriveText()` 函数，按类型 + 申购状态判断）：
  - **ETF** → `实物申赎`（场内 ETF 多为实物申赎，需用一篮子股票，不按现金 T+1 计）
  - **LOF** → `T+2`（场内 LOF 申购份额约两个交易日后到账可卖）
  - 若该基金 `暂停申购` → 显示 `—`（不适用）
- **说明**：这是通用惯例规则，具体以各基金公司公告为准；页脚已注明。

### 功能 5：修复 LOF 数据缺失（重要）
- **背景**：交接时表格里的 LOF 一直是 **0 只**——原代码用 `ak.fund_lof_spot_em()`，但东方财富该接口会**连接重置（RemoteDisconnected）**，且返回字段里**没有 IOPV**，导致 LOF 全部被跳过。
- **修复方式**：在 `app.py` 用自定义 `fetch_lof_list()` 替换它（见第三节「关键技术改动」）。
- **现状**：LOF 现在能正常抓取（约 **370+ 只**），与 ETF 合计约 **1890 只**场内基金。

## 三（补充）、关键技术改动（接手人务必看，别把修复改回去）

### 1. 数据抓取不再依赖 akshare 的 `fund_etf_spot_em` / `fund_lof_spot_em`
这两个接口有两个坑，已绕过：
- **连接被重置**：东方财富对缺请求头的连接会 `RemoteDisconnected`；现统一带 `User-Agent` / `Referer` 请求头。
- **分页坑（最致命）**：东方财富 `clist` 接口**默认每页 100 条，传超大 `pz` 会被静默钳回 100**。
  原代码单次请求只拿到 100 行，导致 ETF 只有 90 只、LOF 只有 81 只（全量应是 1500+ / 370+）。
  **正确做法**：先请求第一页拿到 `data.total`，再按页（`pn=2,3,...`）翻完——代码里 `_fetch_eastmoney_clist()` 已实现。

现在 ETF / LOF 都走 `app.py` 里的：
- `fetch_etf_list()`：字段含 `f441`（IOPV 实时估值），溢价率 = (市价 − IOPV) / IOPV。
- `fetch_lof_list()`：字段含 `f402`（基金折价率）。**LOF 没有 IOPV**，溢价率用折价率反推：`溢价率 = −折价率`。
- 共同底层：`_fetch_eastmoney_clist(fs, fields, page_size=1000)`，带 4 次重试 + 分页。

### 2. 申购状态 / 限额的数据源与解析（新，重点）
- **数据源换成天天基金档案页**：`get_subscribe_map()` → `_fetch_subscribe_one(code)` 抓 `https://fundf10.eastmoney.com/jjfl_<代码>.html`，正则提取「交易状态」和「日累计申购限额」。
  - 为什么不用 AKShare：AKShare 没有返回「开放申购/限大额/暂停」+「限额」的接口（旧代码用的 `fund_open_fund_info_em()` 返回的是历史净值，是错的），所以改抓天天基金档案页。
- **限额数值解析**：`_parse_limit_to_num(limit)` 把 `1000.00元` / `50.00万元` / `2.50亿元` 统一解析成**元**为单位的浮点数，供前端排序；无法解析（如「无」）返回 `None`。
- **只处理 LOF**：ETF 是实物申赎，天天基金的现金申赎状态指向场外联接基金，会误导，故 ETF 单独标注「ETF·实物申赎」，不进真实状态映射。

### 3. 性能优化（首屏从 ~40s 降到 <1s）
- 去掉了 akshare 自带的 **0.5~1.5 秒随机延迟**（翻页时只留 0.1s 间隔）。
- 加了 `_fetch_lock` 抓取锁：避免并发请求各自触发一次完整抓取互相拖慢。
- **启动预热缓存**：`app.py` 在 `__main__` 里先调一次 `get_fund_premium()` 把数据抓好再对外服务，首个访问直接命中缓存（约 90ms 返回）。
- **申购状态后台构建**：LOF 较多且天天基金有节流，`get_subscribe_map()` 首次用后台线程慢慢抓，接口立即返回（先返回空/旧数据），前端每 60 秒自动刷新自然补齐，约 1 分钟填满。

### 4. 缓存与超时
- ETF / LOF 行情缓存 60 秒（`_CACHE_SECONDS`）。
- 申购状态 / 限额缓存 **1 小时**（`_SUBSCRIBE_TTL=3600`）。
- 网页每 60 秒自动刷新，也可点「刷新数据」立即拉取。

## 七、接口说明（给想二次开发的人）

- `GET /`：网页
- `GET /api/premium`：返回全部基金溢价率 JSON（含 `subscribe_info` 申购时间提示）
- `GET /api/refresh`：强制刷新缓存后返回最新数据
- `GET /api/subscribe`：返回 `{基金代码: {label, amount, cls, amount_num}}` 映射
  - `label`：状态文字（如 `限大额·可申购`）
  - `amount`：可申购数额文字（如 `1000.00元`，无则空）
  - `cls`：前端配色类名（`sub-open` / `sub-limit` / `sub-pause` / `sub-etf`）
  - `amount_num`：可申购数额的数值（单位：元，用于排序；无则 `null`）

## 八、常见问题

**Q：页面一直"加载中"或报错？**
A：多半是开了代理/VPN。关掉系统代理后刷新。数据源需要直连。

**Q：申购状态列很多显示 `-`？**
A：两种可能：① 首屏打开时后台还在抓 LOF 申购数据（约数百只，受天天基金节流要几十秒），等约 1 分钟自动刷新会补齐；② 该基金是 ETF，ETF 不取真实状态，列里显示的是「ETF·实物申赎」而非 `-`。若 LOF 长时间仍是 `-`，可能是天天基金临时限流，稍后刷新即可。

**Q：可申购数额列怎么排序？**
A：直接点表头「可申购数额」。后端已把限额统一成「元」数值，万元/亿元也能正确比较；没有限额的（暂停申购、ETF）排到最末尾。

**Q：公网链接（trycloudflare）打不开？**
A：那是临时隧道，每次重启会变，且必须保持本机开机、网站运行、代理关闭。想要固定域名需注册 Cloudflare 配固定隧道。线上固定地址请用 Render 的 `https://fund-premium-site.onrender.com`。

**Q：推送到 GitHub 时弹出要输用户名密码？**
A：正常情况下 `推送更新.bat` 会走 git 凭据弹窗授权。若真要填，用 GitHub 用户名 + 一个**新生成的 PAT**（GitHub → Settings → Developer settings → Personal access tokens，勾 `repo`）当密码填。注意：聊天里出现过的旧 token 已不安全，请新建，不要复用。

**Q：LOF 数量很少或为 0？**
A：早期用 `ak.fund_lof_spot_em()` 会发生连接重置且缺 IOPV，已改成自定义 `fetch_lof_list()`（带请求头 + 分页 + 重试）。若仍异常，多半是东方财富临时限流，等 1 分钟刷新即可（行情有 60s 缓存）。非交易时间部分基金不返回行情也属正常。

**Q：溢价率、到账时间怎么算的？**
A：ETF 溢价率 = (市价 − IOPV) / IOPV，IOPV 来自字段 `f441`；LOF 没有 IOPV，用「折价率 f402」反推，溢价率 = −折价率（折价率为负即溢价）。到账时间：ETF→实物申赎、LOF→T+2、暂停申购→「—」（通用规则，以基金公司公告为准）。
